# [기획서] 로컬 AI 문서 검색 및 지식 관리 서버
## 프로젝트명: LoomQuery

---

## 1. 제품 개요

- **제품명:** LoomQuery
- **제품 정의:** 문서를 업로드하고 AI에게 질문하면 정확한 답변과 출처를 제공하는 프라이버시 우선 로컬 지식 관리 서비스. 외부 시스템 연동을 위한 표준 API도 함께 제공.
- **핵심 타겟:**
  - **Primary:** 사내 문서를 중앙 관리하고 AI 검색을 활용하려는 지식 관리자, 팀 리더, 기획자 등 비개발자
  - **Secondary:** API 연동 및 검색 품질 최적화가 필요한 백엔드/AI 엔지니어
- **배포 형태:** Docker Compose 기반 셀프 호스팅 (로컬/온프레미스)
- **핵심 차별점:** 클라우드에 데이터를 올리지 않는 로컬 우선(Local-first) 아키텍처. 법무, 인사, 재무 등 민감 문서도 안심하고 사용 가능.

---

## 2. 핵심 가치 (Core Value)

1. **프라이버시 우선 AI 검색:** 모든 데이터가 사용자의 환경에 머무름. 클라우드 전송 없이 로컬에서 문서 분석과 AI 검색 수행.

2. **문서 올리고 질문하기:** 드래그 앤 드롭으로 문서를 올리면 AI가 자동으로 분석. 자연어로 질문하면 출처와 함께 답변 제공.
   - 한국형 문서(HWP/HWPX) 포함 다양한 포맷 지원
   - MVP 범위: 텍스트 + 표 추출 (수식/매크로/이미지 OCR은 Phase 2)

3. **누구나 쉽게:** 기술 지식 없이도 바로 사용 가능. 복잡한 설정은 AI가 자동으로 최적화하고, 고급 사용자를 위한 세부 설정은 별도 제공.

---

## 3. 사용자 페르소나 & 역할

| 페르소나 | 주요 관심사 | 핵심 화면 |
|----------|------------|----------|
| 지식 관리자 | 문서 업로드, 폴더 구조, 태그 관리, 문서 검색 | 자료실, 대시보드, AI 질문하기 |
| 팀 리더/기획자 | 빠른 정보 검색, 보고서 요약, 답변 정확성 확인 | AI 질문하기, 대시보드 |
| AI 엔지니어 | 검색 품질 평가, 분석 설정 최적화, 디버깅 | 분석 설정, AI 질문하기(고급) |
| 백엔드 개발자 | API 스펙, 인증, 성능 지표, SDK/코드 스니펫 | API & 연동, AI 질문하기(고급) |

- MVP: 단일 사용자 시나리오 (관리자=사용자)로 시작
- Phase 2: RBAC 기반 역할 분리 도입

## 4. 주요 기능 명세

### A. 문서 관리 (Web UI)

#### A-1. 문서 업로드
- 지원 포맷: PDF, DOCX, XLSX, HTML, TXT, Markdown, HWP, HWPX
- 파일 드래그 앤 드롭 + 클릭 업로드 (두 가지 방식 모두 지원)
- 업로드 즉시 진행 상태 표시: "업로드 완료 → 문서 분석 중 → 검색 준비 완료"
- 제한: 단일 파일 최대 100MB, 동시 처리 최대 3개 (설정 가능)
- 업로드 후 자동 태깅: AI가 문서 내용을 분석하여 카테고리/태그 자동 부여

#### A-2. 문서 분석 설정
- **기본 모드 (권장):** 시스템이 자동으로 최적화 — 사용자는 아무 설정 없이 바로 사용
- **고급 모드 (토글, Secondary 사용자용):**
  - 분석 단위 크기 조정 (128~2048 토큰, 기본 512)
  - AI 엔진 선택
  - 커스텀 전처리 규칙
- **미리보기:** 샘플 20페이지로 사전 실행 → 예상 결과 제시
- **분석 진행률:** SSE 기반 단계별 표시
  - 업로드 완료 → 문서 분석 중 → AI가 문서를 읽고 있어요 → 검색 준비 완료

#### A-3. 문서 상세 보기
- **레이아웃: 원본/관련 구절 2열 동기화 뷰**
  ```
  ┌──────────────────┬───────────────────────┐
  │ 원본 문서 뷰어    │ 관련 구절 목록          │
  │ (선택 구절       │ [구절 1] "계약 조건..." │
  │  하이라이트 연동) │ [구절 2] "이행 기한..." │
  │                  │ ▼ 문서 정보/태그 편집   │
  └──────────────────┴───────────────────────┘
  ```
- 각 구절: 관련도 표시 (높음/보통), 출처 위치
- 태그 체계:
  - 자동 태그: 문서 유형, 날짜, AI가 분류한 카테고리
  - 수동 태그: 부서명, 중요도, 커스텀 레이블
- 대량 처리: 필터링, 검색, 일괄 태그 부여

#### A-4. 폴더 동기화
- 특정 로컬 폴더 감시 및 자동 업데이트
- 변경 감지 → 자동 분석 → 완료 알림
- 삭제된 파일 자동 반영
- 설정 하위에서 감시 대상 폴더 관리

### B. 개발자용 API (Secondary 사용자용)

#### B-1. RESTful API
- `POST /api/ingest` - 원격 문서 업로드 및 분석 실행
- `POST /api/search` - 검색 결과 반환 (관련 구절 + 관련도)
- `POST /api/chat/completions` - OpenAI 호환 포맷 답변 반환
- 에러 응답 표준화: HTTP 상태 코드 + 에러 코드 + 메시지 + Trace ID
  ```json
  {
    "error": {
      "code": "PARSE_UNSUPPORTED",
      "message": "HWP 매크로 포함 문서는 지원되지 않습니다",
      "trace_id": "tr_abc123",
      "recovery": "매크로를 제거 후 재업로드하세요"
    }
  }
  ```

#### B-2. MCP (Model Context Protocol) 지원
- 시스템 자체가 MCP 서버로 동작
- Claude Desktop, Cursor 등 AI 에이전트와 연동
- 별도 프로세스로 실행 (Next.js와 분리)

#### B-3. API Key 및 접근 제어
- 클라이언트별 API Key 발급
- Rate Limiting: 기본 100 req/min (Key별 설정 가능)
- Key 회전(rotation) 및 즉시 무효화 지원
- API 호출 이력 로깅

## 5. UI/UX 화면 구성

### 5-1. 네비게이션 구조
- **좌측 사이드바** + **상단 브레드크럼**
- 접힘(collapsed) 모드 지원
- 글로벌 알림 벨 아이콘 (상단 우측)
- 다크 모드 토글

### 5-2. 화면 목록 (7개)

#### (1) 대시보드 - 개요 허브
- 문서 수, 최근 질문, 저장 용량 등 요약 통계 카드
- 시스템 상태 (AI 엔진 연결, 저장소 용량)
- 최근 활동 피드 (업로드, 분석 완료/실패, 질문 기록)
- 최근 오류 섹션 (바로 이동 CTA)
- 핵심 행동 CTA: "문서 업로드", "질문하기"
- **첫 방문:** 초기 설정 안내 시작 버튼 + 시스템 요구사항 체크 결과

#### (2) 자료실 - 문서 관리
- 폴더 트리 구조 관리 (좌측 패널)
- 문서 목록 (우측) - 상태 표시 (대기/분석중/준비완료/실패)
- 파일 드래그 앤 드롭 업로드
- 각 문서별 분석 진행률 표시
- 문서 클릭 → 문서 상세 보기 진입
- 중복 문서 감지 및 처리 옵션 (덮어쓰기/건너뛰기)
- **첫 방문:** "첫 문서를 업로드하세요" + 샘플 문서 체험 옵션

#### (3) 문서 상세 보기 (자료실 하위)
- 원본/관련 구절 2열 동기화 뷰 (상기 A-3 참조)
- 구절 필터/검색/정렬
- 문서 정보/태그 편집

#### (4) AI 질문하기 - 문서 검색 & 답변
- **2열 레이아웃:**
  ```
  ┌─────────────────────┬──────────────────┐
  │ 대화 인터페이스       │ 참고 문서 패널    │
  │                     │                  │
  │ "무엇이 궁금하세요?" │ 참고한 문서 목록   │
  │                     │ 관련 구절 미리보기  │
  │ 질문/답변            │ 원본 문서 바로가기  │
  │ (출처 인용 포함)     │                  │
  │                     │ [고급] 검색 설정   │
  │                     │ · 참고 문서 수     │
  │                     │ · 답변 스타일      │
  └─────────────────────┴──────────────────┘
  ```
- 자료실 선택 드롭다운
- 인용 표시: 답변 내 인라인 각주 + 하단 참고문헌 목록
- 인용 클릭 → 문서 상세 보기 해당 위치로 딥링크
- 답변 피드백: "도움이 됐어요 / 아니에요" 버튼
- 신뢰도 레이블: "높은 확신" / "참고용" / "확인 필요" (수치 대신 범주형)
- **고급 (토글, Secondary 사용자용):**
  - 검색 파라미터 조정 (참고 문서 수, 답변 스타일)
  - A/B 비교: 설정 변경 후 같은 질문 응답 비교
  - 디버그 패널 (검색된 구절, 관련도, 응답 시간)
  - API 코드 스니펫 내보내기 (curl / Python / JavaScript)
- **첫 방문:** "자료실에 문서를 먼저 업로드하세요" + 예시 질문 카드
  - "이 계약서의 핵심 조건은 뭐야?"
  - "신입사원 온보딩 절차가 어떻게 되나요?"
  - "휴가 정책에 대해 알려주세요"

#### (5) API & 연동 - 개발자 포털 (Secondary 사용자용)
- API Key 목록 및 관리 (생성/삭제/회전/사용량 조회)
- 내장 Swagger/OpenAPI 문서
- MCP 서버 연결 상태 및 설정
- Quick Start Guide (curl / Python / JavaScript 코드 스니펫)
- Webhook 설정 (분석 완료, 오류 발생 시 외부 알림)
- Rate Limit 현황 실시간 조회
- **첫 방문:** "첫 API Key를 발급하세요" + Quick Start 가이드

#### (6) 활동 기록 - 이력 & 로그
- 질문/답변 이력 (시간순, 검색 가능)
- 문서 변경 이력 (업로드/수정/삭제)
- 분석 실행 이력 (성공/실패/소요시간)
- 오류 로그 (필터: 심각도, 유형, 시간범위)
- CSV/JSON 내보내기

#### (7) 설정 - 시스템 설정
- **AI 엔진:** AI 엔진 선택 (로컬 Ollama / OpenAI API) — "기본(권장)" 레이블
- **저장소:** 연결 상태 확인
- **폴더 동기화:** 감시 대상 폴더 관리
- **시스템 정보:** 버전, 저장소 사용량, 서비스 상태
- **표시 설정:** 다크 모드, 언어(한국어)
- **고급 설정 (Secondary 사용자용):**
  - 분석 기본값 조정
  - 벡터 DB 설정 (ChromaDB 기본 / Qdrant 옵션)
  - 로그 레벨 설정

## 6. 핵심 사용자 플로우

### 플로우 1: 처음 시작하기 (온보딩)
```
첫 접속 → 환영 화면
→ "LoomQuery에 오신 것을 환영합니다! 문서를 올리고, AI에게 질문하세요."
→ ① AI 엔진 연결 확인 (자동 점검, 문제 시 안내)
→ ② 샘플 문서 업로드 (한국어 포함, 선택사항)
→ ③ "문서 분석 중..." → "검색 준비 완료!"
→ ④ 첫 질문 체험 (예시 질문 카드 제공)
→ 완료! 대시보드로 이동
```

### 플로우 2: 문서 업로드 & 검색
```
자료실 → 문서 드래그 앤 드롭
→ "문서를 분석하고 있어요..." (진행률 표시)
→ "검색 준비 완료!" → "질문하기" CTA
→ AI 질문하기에서 자연어로 질문
→ 답변 + 출처 인용 확인
→ 출처 클릭 → 원본 문서 해당 위치 확인
```

### 플로우 3: 답변 품질 개선 (고급)
```
AI 질문하기 → 답변 확인
→ "도움이 안 됐어요" 피드백
→ 참고 문서 패널에서 관련 구절 확인
→ 인용 클릭 → 문서 상세 보기
→ [고급] 검색 설정 조정 (참고 문서 수, 답변 스타일)
→ A/B 비교로 최적 설정 탐색
```

### 플로우 4: 문제 해결
```
대시보드 또는 알림으로 오류 감지
→ 활동 기록에서 오류 상세 확인
→ 오류 유형별 안내:
  - 분석 실패: "이 문서는 분석하지 못했어요. 다시 시도하거나 다른 형식으로 변환해보세요"
  - AI 엔진 연결 오류: "AI 엔진과 연결이 끊겼어요. 설정에서 연결 상태를 확인해주세요"
  - 시간 초과: "문서가 너무 커서 시간이 오래 걸리고 있어요. 잠시 후 다시 확인해주세요"
→ 실패 문서 일괄 재시도
```

## 7. 오류 처리 정책

### 오류 분류 체계
| 등급 | 설명 | UI 표현 | 예시 |
|------|------|---------|------|
| 안내 | 일부 기능 제한, 자동 복구 가능 | 노란색 토스트 | 느린 응답, 일부 구절 누락 |
| 오류 | 작업 실패, 재시도 가능 | 빨간색 토스트 + 안내 | 분석 실패, AI 엔진 시간 초과 |
| 심각 | 시스템 사용 불가 | 전체 화면 배너 | 저장소 연결 끊김, 디스크 부족 |

### 오류 메시지 원칙
- 기술 용어 대신 일상 언어 사용
- "무엇이 잘못되었는지" + "어떻게 해결할 수 있는지" 함께 안내
- 다음 행동을 명확히 제시 (재시도 버튼, 설정 이동 링크 등)

### 부분 성공 정책
- 대량 문서 처리 시 부분 성공 허용
- 성공/실패 분리 표시 + 실패 항목 "재시도 대기" 자동 격리
- 실패 문서별 원인 + 해결 방법 안내

### 장시간 작업 피드백
- 진행률 바 + 예상 잔여 시간 + 취소 버튼
- 중단 지점부터 재개(resumable) 지원

## 8. 접근성 및 반응형

- **최소 지원 해상도:** 1280px 이상 데스크톱
- **반응형:** 대시보드만 태블릿 반응형 (요약 통계 확인용)
- **접근성 목표:** WCAG 2.1 AA
  - 키보드 네비게이션 (Tab, Enter, Escape)
  - 스크린 리더 지원 (ARIA 역할, aria-live)
  - 색상 대비 4.5:1 이상
  - 상태 전달: 색상 + 텍스트 + 아이콘 중복 표현
- **다크 모드:** TailwindCSS `dark:` 클래스로 초기부터 지원
- **언어:** 한국어 기본 (i18n 구조는 초기부터 적용, 메시지 키 분리)

## 9. 기술 스택

| 계층 | 기술 | 비고 |
|------|------|------|
| Frontend | Next.js + TailwindCSS + shadcn/ui | SSR 초기 로딩 최적화 |
| Frontend 상태 | Zustand 또는 Jotai | 경량 상태 관리 |
| API / Backend | Next.js API Routes (Route Handlers) | 라우팅 + 인증 + Rate Limiting + 비즈니스 로직 |
| RAG Core | Mastra (`@mastra/rag`) | TypeScript 네이티브, 9가지 청킹 전략, 리랭킹 내장 |
| 문서 파싱 | pdf-parse + mammoth + exceljs | PDF/DOCX/XLSX 개별 최적 라이브러리 조합 |
| 실시간 통신 | Server-Sent Events (SSE) | 분석 진행률 스트리밍 |
| 작업 큐 | BullMQ + Redis | 비동기 문서 분석 작업 (장시간 파싱/임베딩) |
| Vector Database | ChromaDB (기본, 임베디드) | Qdrant는 고급 옵션 |
| 임베딩 | Ollama (로컬) / OpenAI API (클라우드) | Mastra 내장 provider 활용 |
| LLM 스트리밍 | Vercel AI SDK (`ai`) | useChat, streamText 등 UI 스트리밍 |
| MCP Server | 별도 프로세스 (stdio) | Next.js와 분리 실행 |
| 관측성 | OpenTelemetry 구조화 로그 | Trace ID 기반 디버깅 |
| 컨테이너 | Docker Compose | Next.js + ChromaDB + Redis |
